Steganography comes from the greek words "stegan√≥s", which means "covered", and "-graphia", which means writing. The goal of steganography is to find objects where to hide critical messages.
Pictures are a perfect example of objects where a message can be written without being easily noticed.
This program takes a picture and a message and writes this message inside the image, using characters to modify the intensity values of the picture's blue channel. The human eye is more sensitive to variations in both the red and green channel, while it isn't that good ad noticing small differences in the blue one.

As an example, take the character "a". Its binary value (in UTF-8 encoding) is 01100001, which is exactly 8 bits (a byte). The program takes the first 8 pixels of the picture and, in particular, only the blue channel. A blue pixel contains a binary number which is exactly 8 bits; let's say that this number is 01100111, which is 103 in base 10. Then, to hide the first binary digit of "a", which is 0, we substitute the last 1 of the pixel's value with a 0; now, the pixel's value is 01100110, which is 102 in decimal. Then, we take the next pixel, read its binary value and substitute the last digit with 1. Then again with 1, then with 0 (four times) and, lastly, with a 1. By altering by a small amount the value of each pixel (+/- 1), the program can easily write text characters inside an image.
To read the message contained in a picture, the process is reversed. You basically load the last binary digit of each blue pixel in the image, put them together and convert them to UTF-8 characters.
